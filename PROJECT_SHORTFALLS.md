## 🚨 核心问题 Top 5

### 1. **缺失的前端应用** ❌
- `apps/dsa-web/` 与 `apps/dsa-desktop/` 目录**不存在**
- README 中承诺的"Web 工作台"与"Electron 桌面端"无对应代码
- **影响**：文档失真，用户期望与实际差异大

### 2. **代码臃肿难以维护** 📦
- `src/notification.py` (~83KB) + `data_provider/base.py` (~96KB)
- 职责混杂，外部依赖深度耦合
- **影响**：新增功能风险高，改 bug 容易引入新 bug

### 3. **测试覆盖不完整** 🧪
- 60+ 测试文件，但**关键路径缺失**：
  - ❌ 主 Pipeline (`src/core/pipeline.py`)
  - ❌ 股票分析服务 (`src/services/stock_analyzer.py`)
  - ❌ API 端到端测试
  - ❌ 数据源 Fallback 链路
- **影响**：回归风险，无法保证核心流程稳定

### 4. **异步化不彻底** ⚡
- 搜索 Provider（Bocha、Brave、Exa、MiniMax）仍同步
- Pipeline 中存在 `asyncio.to_thread` 阻塞点
- **影响**：并发查询性能 20-40% 下降

### 5. **配置管理过度复杂** ⚙️
- 4 层配置共存：`.env` / 多渠道 / YAML / 高级路由
- 用户无法选择最适合方案
- **影响**：onboarding 难度高，错误率高
